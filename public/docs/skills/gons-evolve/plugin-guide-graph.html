Standalone vs Plugin

Standalone (.claude/)

📁 위치: .claude/skills/
💬 호출: /hello
👤 범위: 개인/프로젝트 전용
🔧 공유: 수동 복사
⚡ 적합: 빠른 실험, 개인 워크플로우
→

Plugin (패키지)

📦 위치: plugin-dir/
💬 호출: /name:hello
👥 범위: 팀/커뮤니티 공유
🏪 공유: Marketplace 설치
📋 적합: 버전 관리, 배포
전략: .claude/에서 먼저 실험 → 검증 후 Plugin으로 변환 → Marketplace로 배포

Plugin 디렉토리 구조

핵심 규칙: plugin.json만 .claude-plugin/ 안에! 나머지 전부 Plugin 루트에 배치!
📦 my-plugin/
📋 .claude-plugin/plugin.json ONLY HERE 매니페스트
⚡ skills/ Optional 스킬 디렉토리
💬 commands/ Legacy 커맨드 (skills/ 권장)
🤖 agents/ Optional 에이전트 정의
🪝 hooks/hooks.json Optional 이벤트 핸들러
📜 scripts/ Optional 유틸리티 스크립트
🔌 .mcp.json Optional MCP 서버 설정
🧠 .lsp.json Optional LSP 서버 설정
⚙️ settings.json Optional 기본 설정

📋 .claude-plugin/plugin.json (매니페스트)

Plugin의 유일한 필수 파일. name 필드만 필수이며, 네임스페이스로 사용됩니다.

{ "name": "my-plugin", ← 필수! 네임스페이스 "version": "1.0.0", ← 시맨틱 버전 "description": "설명", ← Marketplace 표시 "author": { "name": "..." }, "keywords": ["tag1", "tag2"] }

⚡ skills/ (스킬)

Plugin의 핵심. skills/<name>/SKILL.md 구조. 호출: /plugin-name:skill-name

skills/ ├── code-review/ │ ├── SKILL.md ← 필수 │ ├── reference.md ← 보조 파일 │ └── scripts/ ← 스크립트 └── deploy/ └── SKILL.md

💬 commands/ (레거시)

skills/와 동일 기능. 단일 .md 파일로 스킬 정의. 새 프로젝트에서는 skills/ 권장.

🤖 agents/ (에이전트)

Markdown 파일로 정의. YAML frontmatter (name, description) + 시스템 프롬프트 본문.

/agents에서 확인, Claude가 자동 호출 또는 사용자가 수동 호출.

🪝 hooks/hooks.json (훅)

이벤트(PostToolUse, SessionStart 등)에 반응하는 핸들러. 3가지 타입: command, prompt, agent

스크립트 경로에는 반드시 ${CLAUDE_PLUGIN_ROOT} 사용!

📜 scripts/ (유틸리티)

훅이나 스킬에서 참조하는 스크립트. chmod +x 필수! shebang (#!/bin/bash) 확인.

🔌 .mcp.json (MCP 서버)

외부 도구 연동. Plugin 활성화 시 자동 시작. plugin.json에 인라인 정의도 가능.

🧠 .lsp.json (LSP 서버)

언어 서버 설정. 실시간 진단, 코드 네비게이션 제공. LSP 바이너리는 별도 설치 필요!

⚙️ settings.json (기본 설정)

현재 agent 키만 지원. Plugin의 에이전트를 메인 스레드로 활성화.

Plugin 생성 4단계

1

디렉토리 생성

Plugin 루트와 매니페스트 디렉토리 만들기

mkdir -p my-plugin/.claude-plugin
2

plugin.json 작성

name 필드만 필수. 이것이 네임스페이스가 됩니다.

{ "name": "my-plugin", "description": "내 플러그인", "version": "1.0.0" }
name이 "my-plugin"이면 스킬 호출은 /my-plugin:skill-name
3

컴포넌트 추가

skills/, agents/, hooks/ 등 필요한 구성 요소 추가

mkdir -p my-plugin/skills/hello # skills/hello/SKILL.md 작성 --- name: hello description: 인사 스킬 --- 사용자에게 인사하세요.
모든 디렉토리는 Plugin 루트에! .claude-plugin/ 안에 넣지 않기!
4

테스트 & 배포

로컬 테스트 후 Marketplace로 배포

# 로컬 테스트 claude --plugin-dir ./my-plugin # Claude Code 안에서 실행 /my-plugin:hello

구성 요소 상세

SKILL.md Frontmatter 필드
namestring
스킬 이름 (소문자, 숫자, 하이픈, 최대 64자). 생략 시 디렉토리명 사용
descriptionstring
Claude가 자동 호출 판단에 사용. 권장 필드
disable-model-invocationboolean
true → 사용자 수동 호출만 가능 (deploy 등에 적합)
user-invocableboolean
false → / 메뉴에서 숨김. Claude만 호출 가능
allowed-toolsstring
허용 도구 제한. 예: Read, Grep, Glob
modelstring
실행 모델 지정 (haiku, sonnet 등)
contextstring
"fork" → 서브에이전트에서 격리 실행
agentstring
context: fork 시 에이전트 타입 (Explore, Plan 등)
argument-hintstring
자동완성 힌트. 예: [issue-number]
호출 제어 매트릭스
Frontmatter 설정 사용자 호출 Claude 자동 호출 description 로드
기본값 O O 항상
disable-model-invocation: true O X 미로드
user-invocable: false X O 항상
인수 치환 변수
$ARGUMENTS
$ARGUMENTS[N]
$0, $1, $2 ...
${CLAUDE_SESSION_ID}

에이전트 정의 구조
# agents/security-reviewer.md --- name: security-reviewer description: 보안 취약점 전문 검토 에이전트 --- 보안 리뷰 전문가로서, OWASP Top 10을 검토합니다. 입력 검증, 인증, 권한 체크를 중점 분석합니다.
통합 방식: /agents에서 확인 가능 | Claude가 작업 맥락에 따라 자동 호출 | 사용자가 수동 호출 가능
settings.json 연동: {"agent": "security-reviewer"}로 메인 에이전트 설정 가능

훅 타입 3가지
command
셸 명령/스크립트 실행. ${CLAUDE_PLUGIN_ROOT}로 경로 참조
prompt
LLM에 프롬프트 평가. $ARGUMENTS로 컨텍스트 전달
agent
에이전틱 검증기. 도구 사용 가능한 복잡한 검증 작업
MCP 서버 설정 (.mcp.json)
{ "mcpServers": { "my-db": { "command": "${CLAUDE_PLUGIN_ROOT}/servers/db-server", "args": ["--config", "${CLAUDE_PLUGIN_ROOT}/config.json"], "env": { "DB_PATH": "${CLAUDE_PLUGIN_ROOT}/data" } } } }
Plugin 활성화 시 자동 시작 Claude 도구로 자동 통합 plugin.json에 인라인 정의 가능
LSP 서버 설정 (.lsp.json)
{ "go": { "command": "gopls", ← 필수 (PATH에 있어야 함) "args": ["serve"], "extensionToLanguage": { ← 필수 ".go": "go" }, "transport": "stdio", ← 선택 (기본: stdio) "restartOnCrash": true, ← 선택 "maxRestarts": 3 ← 선택 } }
주의: LSP 바이너리는 Plugin에 포함되지 않습니다. 사용자가 별도로 설치해야 합니다.
제공 기능
실시간 진단 (에러/경고) 코드 네비게이션 (go to definition) 타입 정보 & 문서

Local (개인/프로젝트)

gitignore 대상. 이 프로젝트에서만, 이 사람만

# 기본 (user) claude plugin install my-plugin@marketplace # 프로젝트 범위 (팀 공유, git 추적) claude plugin install my-plugin@marketplace --scope project # 로컬 범위 (gitignore) claude plugin install my-plugin@marketplace --scope local
