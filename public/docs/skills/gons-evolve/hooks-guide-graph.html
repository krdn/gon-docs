Claude Code Hooks

생명주기 자동화 시스템 — 이벤트, 타입, 데이터 흐름, 설정까지

🔄 Lifecycle Timeline
Claude Code의 생명주기에서 Hook이 발동하는 18개 이벤트. 왼쪽은 사용자/세션 이벤트, 오른쪽은 도구/에이전트 이벤트입니다.
SESSION START
SessionStart
세션 시작 또는 재개 시 발동. 컨텍스트 주입, 환경 초기화에 사용
matcher: startup | resume | clear | compact
ConfigChange
설정/스킬 파일이 외부에서 변경될 때 발동. 감사 로그나 변경 차단
matcher: user_settings | project_settings | skills
USER INPUT
UserPromptSubmit
프롬프트 제출 시, Claude가 처리하기 전에 발동. 입력 검증, 컨텍스트 추가
matcher: 미지원 (항상 발동)
TOOL EXECUTION
PreToolUse
도구 호출 실행 전에 발동. 차단(deny), 허용(allow), 확인(ask) 가능
matcher: Bash | Edit|Write | mcp__.*
PermissionRequest
권한 대화상자 표시 시 발동. 자동 승인/거부 로직. 비대화형 모드(-p) 미작동
matcher: 도구 이름 (Bash, Edit 등)
PostToolUse
도구 호출 성공 후 발동. 포맷팅, 로깅 등. 이미 실행된 작업은 되돌릴 수 없음
matcher: 도구 이름 (Bash, Edit|Write 등)
PostToolUseFailure
도구 호출 실패 후 발동. 에러 로깅, 대체 경로 안내
matcher: 도구 이름
SUBAGENTS
SubagentStart
서브에이전트 생성(스폰) 시 발동. 추적, 초기 설정
matcher: Bash | Explore | Plan | 커스텀
SubagentStop
서브에이전트 종료 시 발동. 정리 작업, 결과 기록
matcher: 에이전트 타입
TeammateIdle
팀 에이전트가 유휴 상태로 전환될 때 발동. 작업 재배정
matcher: 미지원 (항상 발동)
TaskCompleted
태스크가 완료로 표시될 때 발동. 자동 검증
matcher: 미지원 (항상 발동)
WORKTREE & COMPACT
WorktreeCreate
Worktree 생성 시 발동. 기본 git 동작을 대체
matcher: 미지원 (항상 발동)
WorktreeRemove
Worktree 제거 시 (세션 종료 또는 서브에이전트 완료)
matcher: 미지원 (항상 발동)
PreCompact
컨텍스트 압축 직전에 발동. 상태 저장
matcher: manual | auto
NOTIFICATION & END
Notification
Claude Code가 알림을 보낼 때 발동. 데스크톱 알림 연동

Stop
Claude 응답 완료 시 발동. 완료 검증, 추가 작업 지시. 인터럽트 시 미발동
matcher: 미지원 (항상 발동)
SessionEnd
세션 종료 시 발동. 정리, 로깅
matcher: clear | logout | prompt_input_exit | other
🔧 Hook Types
4가지 훅 타입의 특징과 선택 기준. 각 타입이 어떻게 동작하고 언제 사용하는지 한눈에 파악합니다.
💻
command 기본
셸 명령을 직접 실행합니다. 대부분의 훅에서 사용하는 기본 타입. stdin으로 JSON을 받고, exit code와 stdout/stderr로 결과를 반환합니다.
stdin으로 이벤트 JSON 수신
exit 0 = 허용, exit 2 = 차단
stdout → Claude 컨텍스트 주입
/hooks 메뉴에서 설정 가능
기본 타임아웃: 10분
🌐
http 외부 연동
HTTP POST로 이벤트 데이터를 외부 엔드포인트에 전송합니다. 팀 공유 감사 로그, 클라우드 함수 연동 등에 적합합니다.
JSON POST (command stdin과 동일 데이터)
응답 본문으로 결과 반환 (JSON)
헤더에서 $VAR 환경변수 보간
allowedEnvVars로 변수 허용 목록 지정
JSON 직접 편집만 가능 (/hooks 메뉴 미지원)
🧠
prompt LLM 판단
LLM(기본 Haiku)에게 프롬프트를 보내 1회 판단을 요청합니다. 규칙으로 표현하기 어려운 주관적 조건 검사에 적합합니다.
ok: true (허용) / ok: false (차단 + reason)
model 필드로 모델 변경 가능
입력 데이터만으로 판단 (도구 사용 불가)
단일 턴 LLM 호출
🤖
agent 서브에이전트
서브에이전트를 스폰하여 파일 읽기, 코드 검색, 명령 실행 등 도구를 사용한 검증을 수행합니다. 코드베이스 실제 상태 확인에 적합합니다.
ok: true / ok: false (prompt 훅과 동일 형식)
파일 읽기, 코드 검색, 명령 실행 가능
최대 50턴 도구 사용
기본 타임아웃: 60초 (설정 가능)

📊 Data Flow
Hook이 이벤트 데이터를 받고, 처리하고, 결과를 반환하는 전체 데이터 흐름을 단계별로 설명합니다.
1

이벤트 발생

Claude Code의 생명주기에서 특정 시점에 도달하면 이벤트가 발생합니다. 예: Claude가 Bash 도구를 호출하려 할 때 PreToolUse 이벤트가 발동됩니다.

2

Matcher 검사

등록된 훅의 matcher 패턴과 이벤트 데이터를 비교합니다. 정규식(regex) 매칭이며 대소문자를 구분합니다. Matcher가 비어있으면 모든 이벤트에 반응합니다.

// matcher: "Edit|Write" // tool_name이 "Edit" 또는 "Write"일 때만 매칭 "matcher": "Edit|Write" ← 정규식 OR 패턴
3

입력 전달 (stdin JSON)

매칭된 훅에 이벤트 데이터를 JSON으로 전달합니다. 공통 필드(session_id, cwd, hook_event_name) + 이벤트별 고유 필드가 포함됩니다.

{ "session_id": "abc123", // 세션 고유 ID "cwd": "/home/gon/project", // 작업 디렉토리 "hook_event_name": "PreToolUse", // 이벤트 이름 "tool_name": "Bash", // 도구 이름 "tool_input": { // 도구 인자 "command": "npm test" } }
4

훅 실행 (병렬)

매칭된 모든 훅이 병렬로 실행됩니다. 동일한 명령은 자동으로 중복 제거됩니다. 각 훅 타입에 따라 실행 방식이 다릅니다.

command
셸 명령 실행, stdin으로 JSON 수신
http
JSON POST 전송, 응답 본문 수신
prompt
LLM 1회 호출, ok/reason 반환
agent
서브에이전트 스폰, 도구 사용 검증
5

결과 반환 (Exit Code / JSON)

훅의 실행 결과에 따라 Claude Code가 다음 행동을 결정합니다. exit code가 기본이며, 더 세밀한 제어가 필요하면 JSON 출력을 사용합니다.

0
허용 (Proceed)
작업 진행. stdout은 Claude 컨텍스트에 추가됨
2
차단 (Block)
작업 차단. stderr가 Claude에게 피드백됨
*
오류 (Error)
작업 진행. stderr는 로그에만 기록 (Ctrl+O)
고급: 구조화된 JSON 출력
{ "hookSpecificOutput": { "hookEventName": "PreToolUse", "permissionDecision": "deny", // allow | deny | ask "permissionDecisionReason": "rg 사용 권장" } }
exit 2와 JSON을 혼합하면 안 됩니다. exit 2일 때 JSON은 무시됩니다.
🎯 Matcher System
Matcher는 훅이 발동되는 조건을 필터링합니다. 각 이벤트 타입별로 matcher가 필터하는 대상이 다릅니다.
🛠 도구 이름으로 필터 4 이벤트
PreToolUse Bash, Edit|Write, mcp__.*
PostToolUse Edit|Write, Bash
PostToolUseFailure Bash, mcp__github__.*
PermissionRequest Bash, Edit, Write
MCP 도구 이름 규칙: mcp__<서버>__<도구> 형식.
예: mcp__github__search_repositories, mcp__filesystem__read_file
🔄 세션 상태로 필터 2 이벤트
SessionStart startup, resume, clear, compact
SessionEnd clear, logout, prompt_input_exit, other
🤖 에이전트 타입으로 필터 2 이벤트
SubagentStart Bash, Explore, Plan, 커스텀 이름
SubagentStop Bash, Explore, Plan, 커스텀 이름
🔍 기타 필터 대상 3 이벤트
Notification permission_prompt, idle_prompt, auth_success
PreCompact manual, auto
ConfigChange user_settings, project_settings, skills

실전에서 바로 사용할 수 있는 훅 설정 예제. 각 카드에 이벤트, matcher, 타입, 저장 위치와 동작 흐름이 표시됩니다.
🔔

데스크톱 알림

Claude가 입력을 기다릴 때 알림

Notification matcher: * command 전역
터미널을 계속 주시하지 않아도 Claude가 권한을 요청하거나 유휴 상태일 때 데스크톱 알림을 받습니다.
"Notification": [{ "matcher": "", "hooks": [{ "type": "command", "command": "notify-send 'Claude' '입력 대기 중'" }] }]
알림 발생 → notify-send 실행 → 데스크톱 알림 표시
✨

자동 코드 포맷팅

파일 편집 후 Prettier 자동 실행

PostToolUse Edit|Write command 프로젝트
Claude가 파일을 편집할 때마다 자동으로 Prettier를 실행하여 일관된 코드 스타일을 유지합니다.
"PostToolUse": [{ "matcher": "Edit|Write", "hooks": [{ "type": "command", "command": "jq -r '.tool_input.file_path' | xargs npx prettier --write" }] }]
Edit/Write 완료 → jq로 경로 추출 → Prettier 실행
🛡

보호 파일 편집 차단

.env, .git/ 등 민감 파일 보호

PreToolUse Edit|Write command (script) 프로젝트
외부 스크립트에서 파일 경로를 검사하고, 보호 패턴에 매칭되면 exit 2로 차단합니다. Claude에게 차단 이유가 피드백됩니다.
#!/bin/bash INPUT=$(cat) FILE=$(echo "$INPUT" | jq -r '.tool_input.file_path') # 보호 패턴 검사 if [[ "$FILE" == *".env"* ]]; then echo "Blocked: .env" >&2 exit 2 # 차단! fi exit 0 # 허용
Edit 시도 → 패턴 매칭 → exit 2 차단 → Claude 피드백
🔁

컨텍스트 재주입

압축 후 핵심 정보 복원

SessionStart compact command 프로젝트
컨텍스트 압축으로 손실된 핵심 정보를 자동으로 다시 주입합니다. stdout이 Claude 컨텍스트에 추가됩니다.
"SessionStart": [{ "matcher": "compact", "hooks": [{ "type": "command", "command": "echo 'bun 사용, 커밋 전 bun test 실행'" }] }]
컨텍스트 압축 → SessionStart(compact) → stdout → 컨텍스트
🧠

작업 완료 검증 (LLM)

Prompt 훅으로 완료 여부 판단

Stop prompt 프로젝트
Claude가 응답을 완료하면 LLM이 모든 작업이 끝났는지 판단합니다. 미완료 시 reason이 Claude에게 다음 지시로 전달됩니다.
"Stop": [{ "hooks": [{ "type": "prompt", "prompt": "모든 작업이 완료되었는지 확인. 미완료 시 {\"ok\": false, \"reason\": \"남은 작업\"}" }] }]
Claude 응답 완료 → LLM 판단 → ok:false → 계속 작업
🤖

테스트 통과 검증 (Agent)

서브에이전트가 테스트 실행 후 확인

Stop agent 프로젝트
Claude 응답 완료 시 서브에이전트가 실제로 테스트를 실행하여 통과 여부를 확인합니다. 파일 읽기, 명령 실행이 가능합니다.
"Stop": [{ "hooks": [{ "type": "agent", "prompt": "테스트 스위트 실행 후 결과 확인", "timeout": 120 }] }]
Claude 완료 → Agent 스폰 → 테스트 실행 → ok/reason

🧩 컴포넌트 내장

Plugin hooks/ | Skill/Agent frontmatter
플러그인, 스킬, 에이전트에 훅을 내장합니다. 컴포넌트가 활성화될 때만 작동합니다.
공유 가능 활성화 시

설정 팁

• /hooks 메뉴로 추가한 훅 → 즉시 반영
• 설정 파일 직접 편집 → /hooks 메뉴에서 확인하거나 세션 재시작 필요
• 모든 훅 비활성화: 설정에 "disableAllHooks": true 추가 또는 /hooks 메뉴 하단 토글
• HTTP 훅은 /hooks 메뉴에서 설정 불가 → JSON 직접 편집 필요
🚨 Troubleshooting
자주 발생하는 문제와 해결 방법. 각 카드에 증상, 원인, 해결책이 정리되어 있습니다.
🚫

훅이 발동하지 않음

설정한 훅이 실행되지 않는 경우. Matcher 패턴 불일치, 잘못된 이벤트 타입, 비대화형 모드에서 PermissionRequest 사용 등이 원인일 수 있습니다.
✓
/hooks에서 올바른 이벤트 아래 표시 확인
Matcher 대소문자 일치 확인
PermissionRequest → PreToolUse 변경 (비대화형)
❌

"hook error" 메시지

스크립트가 예기치 않게 비정상 종료. "command not found", "jq not found", 실행 권한 없음 등.
✓
수동 테스트: echo '{"tool_name":"Bash"}' | ./hook.sh
chmod +x로 실행 권한 부여
절대 경로 또는 $CLAUDE_PROJECT_DIR 사용
🔄

Stop 훅 무한 루프

Claude가 멈추지 않고 계속 작업. Stop 훅이 자기 자신을 재발동시키는 경우.
✓
stop_hook_active 필드 확인:
if [ "$(jq -r '.stop_hook_active')" = "true" ]; then exit 0; fi
📋

JSON 파싱 실패

셸 프로필(~/.zshrc)의 echo 출력이 JSON 앞에 섞여 파싱 오류 발생.
✓
