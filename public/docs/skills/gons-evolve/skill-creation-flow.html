→
📝2
SKILL.md 작성
Frontmatter + 마크다운 지시사항
→
📎3
보조 파일 추가
템플릿, 예시, 스크립트 (선택)
→
🧪4
테스트
/skill-name 또는 자연어로 호출
→
🚀5
배포
Git 커밋 또는 플러그인으로 공유

SKILL.md 해부도

SKILL.md
--- name: fix-issue description: Fix a GitHub issue by number argument-hint: [issue-number] disable-model-invocation: true allowed-tools: Bash(gh *), Read, Edit context: fork agent: general-purpose model: sonnet --- # Fix GitHub Issue <!-- 동적 컨텍스트 주입 --> 이슈 정보: !`gh issue view $ARGUMENTS` ## 절차 1. 이슈 $ARGUMENTS 분석 2. 관련 코드 탐색 3. 수정 구현 4. 테스트 작성 5. 커밋 생성
📋
name
/fix-issue로 호출되는 스킬 이름. 소문자, 숫자, 하이픈만 허용
💬
description
Claude가 자동 호출 판단 시 참조하는 설명. 자연어 키워드 포함 권장
🔒
disable-model-invocation
true → Claude 자동 호출 차단. 배포/커밋 등 부작용 있는 스킬에 필수
🔧
allowed-tools
스킬 실행 시 승인 없이 사용 가능한 도구. 와일드카드 지원: Bash(gh *)
🔀
context: fork + agent
격리된 서브에이전트에서 실행. Explore(읽기전용), Plan(설계), general-purpose(전체)
⚡
$ARGUMENTS & !`cmd`
$ARGUMENTS: 사용자 인자 치환. !`cmd`: 셸 명령 실행 후 결과 주입 (전처리)

스킬 설정 의사결정 트리

❓ 부작용(side-effect)이 있는가?
배포, 커밋, 메시지 전송, DB 변경 등
Yes No
🔒 disable-model-invocation: true
사용자만 /name으로 수동 호출
❓ 사용자가 직접 호출할 일이 있는가?
Yes No
❓ 대화 이력이 필요한가?
Yes No
✅ 기본값 (인라인)
별도 설정 불필요
✅ context: fork
격리 실행, 무거운 작업에 적합
📚 user-invocable: false
배경 지식 전용, / 메뉴에서 숨김

호출 제어 매트릭스

설정 사용자 호출 Claude 자동 호출 컨텍스트 로딩 사용 예시
(기본값) ✓ 가능 ✓ 가능 설명만 항상 로드 코드 설명, 리팩토링 가이드
disable-model-invocation: true ✓ 가능 ✗ 불가 컨텍스트에 미포함 배포, 커밋, 메시지 전송
user-invocable: false ✗ 불가 ✓ 가능 설명 항상 로드 레거시 시스템 컨텍스트, 컨벤션

변수 치환 & 동적 주입

$ARGUMENTS
전체 인자
/skill-name 뒤에 입력한 모든 텍스트
/fix-issue 123 → $ARGUMENTS = "123"
$ARGUMENTS[N] / $N
위치별 인자 (0-based)
공백으로 구분된 N번째 인자
/migrate A B C → $0="A" $1="B" $2="C"
${CLAUDE_SESSION_ID}
세션 ID
현재 세션 고유 식별자. 로깅에 유용
logs/${CLAUDE_SESSION_ID}.log
!`command`
동적 컨텍스트 주입
셸 명령 실행 → 결과를 스킬에 삽입 (전처리)
!`gh pr diff` → PR diff 내용 삽입
