동기화(배포) 시스템

sync.json 매핑 설정을 기반으로 소스에서 ~/.claude/로 자동 배포

sync.sh 명령어
scripts/sync.sh -n
dry-run — 변경 미리보기
scripts/sync.sh
전체 동기화 (확인 후 실행)
scripts/sync.sh --item gon:todo
특정 항목만 동기화
scripts/sync.sh --check
sync.json 유효성 검사
scripts/sync.sh -f
확인 없이 강제 동기화
의존성
jq + rsync

Evolve 자기 개선 흐름

/gon:evolve가 이 저장소를 분석하고 개선하는 전체 사이클

/gon:evolve 전체 사이클
Step 1
인벤토리 수집
sync.json을 파싱하여 모든 스킬, 훅, 에이전트 목록 수집
Step 2
evolve-analyzer 에이전트 스폰
코드베이스 전체를 분석하는 전문 에이전트를 생성. Read, Bash, Grep, Glob 도구 사용
Step 3
코드베이스 분석 → 발견 목록 반환
스킬 품질, 훅 구조, 에이전트 필드, sync 정합성, 문서 최신성 점검. 결과를 JSON으로 반환
Step 4
결과 종합 → 사용자 승인
발견된 이슈를 severity 순(error→warning→info)으로 정리 후 적용 여부 확인
Step 5
개선 적용
승인된 항목에 대해 파일 수정 실행
Step 6
동기화 검증 → 배포
sync.sh dry-run으로 검증 후 ~/.claude/에 배포
Step 7
evolution-log.json 이력 기록
사이클 결과를 append-only 로그 파일에 영구 기록
evolve-analyzer 분석 체크리스트
SQ — 스킬 품질
✓ frontmatter 필수 필드 존재 여부
✓ allowed-tools 목록 정합성
✓ 서브커맨드 문서화 완전성
✓ 에러 처리 섹션 존재
✓ semver 버전 형식 준수
HK — 훅 품질
✓ shebang 존재 여부
✓ 적절한 에러 처리 (set -euo pipefail)
✓ 프로젝트별 훅 체이닝(.local) 지원
✓ 건너뛰기 메커니즘 (환경변수)
✓ 민감 파일 패턴 최신성
