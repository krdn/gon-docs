FSD 레이어 계층 구조
Feature-Sliced Design. 위에서 아래 방향의 단방향 의존성만 허용합니다. eslint-plugin-boundaries로 강제.
app
페이지, 레이아웃, API Routes — 모든 레이어를 import할 수 있는 최상위 레이어
pages layouts API routes [locale]
▼
components
도메인별 UI 컴포넌트 — features, lib, shared만 import 가능. app import 금지.
counseling grades students ui matching
▼
lib
Server Actions, DAL, 유틸리티 — ★ FSD 변형: Server Actions가 feature 서비스 직접 호출
actions/ dal.ts session.ts db/ errors/
▼
features
비즈니스 로직 — lib, shared만 import 가능. app/components import 금지.
ai-engine analysis counseling grade-management matching report
▼
shared
공통 타입, 상수, 검증 스키마 — 의존성 없음. 최하위 레이어.
constants types utils validations errors

'use server'; import { getCurrentTeacher } from '@/lib/dal'; import { ok, fail, type ActionResult } from '@/lib/errors/action-result'; import { logger } from '@/lib/logger'; export async function myAction(input: string): Promise<ActionResult<MyType>> { try { const teacher = await getCurrentTeacher(); // ① 인증 // Zod safeParse()로 입력 검증 // ② 검증 const result = await featureService(input, teacher.id); // ③ 서비스 revalidatePath('/path'); // ④ 캐시 무효화 return ok(result); // ⑤ 성공 반환 } catch (error) { logger.error({ err: error }, 'Failed to do something'); return fail(error instanceof Error ? error.message : '오류가 발생했습니다.'); } }

핵심 인프라 파일
파일 역할
src/lib/dal.ts 인증(verifySession) + RBAC + RLS 컨텍스트. 모든 Server Action의 인증 게이트웨이.
src/lib/session.ts JWT 세션 생성/검증/삭제. 7일 만료, jose HS256 알고리즘.
src/lib/db/client.ts Prisma 싱글톤. pg Pool max:10 연결 풀 관리.
src/lib/errors/action-result.ts ActionResult 패턴 구현. ok(data) / fail(msg) / fieldError(fields) 반환.
src/middleware.ts Edge 미들웨어. 인증 체크 + next-intl i18n 라우팅 + Request ID 주입.
src/features/ai-engine/universal-router.ts LLM 라우팅/페일오버 핵심 로직. 11개 Provider 간 자동 전환.
